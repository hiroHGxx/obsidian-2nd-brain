# レクチャー 6.3 詳細設計書

## 1社ごとのパーソナライズ ― "この会社のHPを踏まえて書き換えて"の威力

---

## ① レクチャーの位置づけ

前のレクチャー6.2では、業種別にDMを一括生成する「量産」の技を習得した。受講者は「テンプレートを回すだけで3パターン一気に作れる」という効率の喜びを手にしている。このレクチャーでは、その量産テンプレートに「ひと手間」加えて、1社ごとに刺さるDMへ変換する応用技を教える。受講者が得る1つのゴールは「テンプレートDMを、相手企業のHP情報を踏まえた個社向けDMに30秒で書き換えられるようになること」。次のセクションS7（7.1）への橋渡しとして、「量産×パーソナライズの合わせ技こそが、明日から使える最強ワークフローだ」という実感を持たせ、最終レクチャーの「明日の朝これだけやって」アクションに自然につなげる。

## ④ ライブ実演の操作手順

```
【実演パート名：HP情報を踏まえたDMパーソナライズ】

＜操作手順＞
1. Claudeの新規チャット画面を表示する（タブ1）
2. タブ2に切り替え、メモ帳で開いた「中部精密工業HP情報.txt」を画面に映す（2秒間）
3. タブ1（Claude）に戻る
4. 事前に用意したプロンプトA全文をクリップボードからチャット入力欄に貼り付ける
   （タイピングせず、Ctrl+Vで貼り付け。貼り付け後、入力欄を上にスクロールしてプロンプト全体を一瞬見せる）
5. 画面右上のストップウォッチアプリで計測開始ボタンをクリック
6. Enterキーで送信
7. Claudeが応答を生成するのを待つ（15〜20秒）
8. 応答が完了したらストップウォッチを停止
9. 出力されたDMを上からゆっくりスクロールして全文を見せる
10. 変更箇所①（件名）をマウスカーソルで指し示す ― 「精密バルブ」等の相手企業固有キーワードが入っていることを確認
11. 変更箇所②（冒頭の一文追加）をマウスカーソルで指し示す ― EV量産ラインのニュースに触れていることを確認
12. 変更箇所③（課題表現の変更）をマウスカーソルで指し示す ― 「精密部品の生産ライン」等、相手の事業に寄せた表現になっていることを確認

＜講師が強調すべきポイント＞
- 「HPをコピペして渡すだけ。特別な指示は要りません。」
- 「手書きで5分、AIなら30秒。しかも毎回安定した品質で出てきます。」
- 「優先度Aの会社だけこのひと手間をかける。全件やる必要はありません。」

＜想定トラブルと対処＞
- Claudeが件名を変更しなかった場合：「件名も変えて」と追加指示を出す実演に切り替え、
  「こういう時は一言追加すれば直ります。AIとの会話は"キャッチボール"です」と解説
- HPに書いてない情報をClaudeが創作した場合（ハルシネーション）：
  「ここ見てください、こんな事業やってないですよね。これがAIの勘違いです。
   だから必ず自分の目で最終チェック。30秒で読めますから、必ずやってください」と注意喚起
- Claudeの応答が遅い場合（30秒以上）：
  「長い文章を書き換えていると少し時間がかかることもあります。
   それでも手書きより圧倒的に速いですよね」と自然にフォロー
```

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示
このレクチャーは「応用編」だが、受講者のテンションは6.2の成功体験で高まっている状態。その勢いを維持しつつ、「もうワンランク上」を見せるワクワク感を演出する。語り口は「すごいでしょ」ではなく「これ、楽じゃないですか？」の共感トーン。パーソナライズの手間を強調しすぎない。「ひと手間」「30秒」「コピペするだけ」を繰り返し、簡単さを印象づける。

### NGワード

| NGワード | 言い換え |
|---------|---------|
| ハルシネーション（初出時以外） | AIの勘違い |
| プロンプトエンジニアリング | 指示の出し方 |
| パーソナライゼーション | 個社向けに書き換える |
| コンテキスト | 相手の情報、背景 |
| API | （この回では登場しない） |
| ナーチャリング | 関係づくり |
| テンプレートリテラシー | （使わない） |

### カメラ・マイク
画面共有のみ。顔出し不要。マイクはピンマイクまたはコンデンサーマイク。ライブ実演中はマウスカーソルの動きが見えるよう、カーソルハイライト（黄色い円）をONにする。

### 編集指示
- 0:50〜1:20のプロンプト貼り付けシーン：貼り付け後にプロンプト全体を一瞬映すが、読ませる必要はない。「詳しくは特典テンプレートに入っています」のテロップを右下に2秒間表示
- 1:25〜1:45のClaude応答待ち：実際に15〜20秒かかる場合は、10秒程度に早送り編集。ストップウォッチの表示は実時間のまま残す（リアリティ）
- 1:45〜2:10の変更箇所指差し：変更箇所3箇所それぞれに赤い下線テロップを追加（「①件名にキーワード追加」「②最新ニュースに触れた一文」「③課題を相手の事業に寄せた」）
- 2:30〜2:45の注意点パート：「AIの勘違い」に初めて言及する箇所。画面に「⚠ 必ず目視チェック」のテロップを大きく表示（3秒間）

### 前後のレクチャーとの接続
- **冒頭（0:00）：** 6.2の成果物（テンプレートDM）を前提にスタートする。「前回作ったあのテンプレート、覚えてますか？」で自然に接続
- **末尾（2:45〜3:00）：** S6全体の振り返り（リスト→整形→訪問計画→DM）を10秒で総括し、S7への橋渡し。「次はこれを"毎週の習慣"に変えます」で締める。S6の最終レクチャーとしてセクション全体の達成感を演出する
