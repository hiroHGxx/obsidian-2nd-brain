# レクチャー 4.2 詳細設計書

## 優先度ランク付け ― AIに"次にアプローチすべき企業"を考えさせる

**レクチャーID:** 4.2  
**レクチャー時間:** 4分  
**セクション:** S4: 営業リストを"戦略的な武器"に仕上げる ― Excel関数なしの自動整形

---

## ① レクチャーの位置づけ

前のレクチャー4.1で、受講者は重複排除・業種フィルタによる「データクレンジング」を体験し、"きれいに整ったリスト"を手に入れた状態にある。しかし、きれいなだけでは営業成果は出ない。本レクチャーでは、そのリストに**優先度A/B/Cのランク**を付け、「次にどの企業からアプローチすべきか」をAIに考えさせることで、リストを"ただの一覧"から"戦略的な武器"へ昇華させる。これがS4セクション全体の到達点であり、受講者が「AIってデータ整理だけじゃなく、判断の補助までしてくれるのか」と気づく転換点になる。次のS5（エリア分け＋訪問計画）では、このランク付き企業リストをもとにエリア別のグループ分けと訪問計画を行うため、本レクチャーの出力がS5全体のインプットとなる。

## ④ ライブ実演の操作手順

### 【実演パート1：データ貼り付けと優先度ランク付け指示】

＜操作手順＞

1. Claudeの画面を開く（新規チャット、何も入力されていない状態）
2. 事前にコピーしておいた15社分の企業リスト（マークダウンテーブル形式）を、チャット入力欄にペーストする
3. ペーストしたデータの下に、改行を1行入れる
4. 以下のプロンプトをテキストファイルからコピーし、データの下に貼り付ける：

```
上記の企業リストに対して、以下の優先基準で営業アプローチの優先度をA・B・Cの3段階でランク付けしてください。

【優先基準】
- 従業員数50名以上 → 高優先
- 設立年が2015年以前（業歴10年以上） → 高優先
- 上記の両方を満たす → 優先度A
- どちらか一方を満たす → 優先度B
- どちらも満たさない → 優先度C

【出力形式】
元の表に以下の2列を追加して、表形式で出力してください。
- 優先度（A / B / C）
- 判定理由（1行で簡潔に）

優先度Aの企業を上に、Cの企業を下に並べてください。
```

5. Enterキーで送信する
6. Claudeが表を生成するのを待つ（5〜10秒）
7. 出力された表をスクロールして全体を見せる
8. 優先度A企業の「判定理由」列を指さし、AIが自動で理由を書いていることを強調する

＜講師が強調すべきポイント＞

- 「15社全部にランクと理由が付きました。手作業でやったら、1社ずつ条件を確認して…15分はかかりますよね。」
- 「"判定理由"が地味にすごい。自分でメモ書くの、めんどくさいですよね。」

＜想定トラブルと対処＞

- 従業員数を数値として正しく認識しない場合（「280名」の「名」が邪魔をする）：「もし数字をうまく読めなかったら、『従業員数の"名"は無視して数値だけで判定して』と追加指示すればOKです」とフォロー
- 優先度の判定が期待と異なる場合：「AIの判定と自分の感覚がズレることもあります。それは次のパートで直します」と自然につなげる

### 【実演パート2：条件カスタマイズの追加指示】

＜操作手順＞

1. 実演パート1のClaudeの出力がそのまま表示されている状態で、続けてチャット入力欄に以下のプロンプトを入力する：

```
ありがとうございます。条件を1つ追加します。
業種が「IT」の企業は、従業員数に関わらず優先度Aに引き上げてください。
理由：当社の新商品はIT企業との相性が良いため。
上記の条件で、もう一度ランク付けし直してください。
```

2. Enterキーで送信する
3. Claudeが再ランク付けした表を生成するのを待つ（5〜10秒）
4. IT企業がAに移動したことを指さして確認する

＜講師が強調すべきポイント＞

- 「こうやって"自分の営業方針"を後から追加できる。会議で方針が変わっても、1行追加するだけでリスト全体が更新されます。」
- 「Excel関数だったら、条件を1個追加するたびにIF文を直して…考えるだけで嫌ですよね。」

＜想定トラブルと対処＞

- 前回の表と構成が変わってしまう場合（列の順番が入れ替わるなど）：「表の形が変わったら、『前回と同じ列順で出力して』と一言添えれば大丈夫です」とフォロー

## ⑤ スライドの指示

### 【スライド1：タイトルスライド】

- **表示テキスト：** AIに"次にどこへ電話すべきか"を考えさせる
- **サブテキスト：** 優先度ランク付け ― リストが"武器"に変わる瞬間
- **ビジュアル要素：** 企業リストのアイコン（表のイメージ）に「A」「B」「C」のラベルが付いている簡易イラスト

- **講師のセリフ（要旨）：** 冒頭「前回きれいにしたリスト、"で、どこから電話すれば？"を解決します」／締め「次はこのリストでエリア分けと訪問計画を作ります」

### 【スライド2：ランク付け基準の図解】

- **表示テキスト：** あなたの"頭の中の基準"をAIに伝える
- **サブテキスト：** （なし）
- **ビジュアル要素：** 左側に吹き出し3つ（「従業員が多い → 案件が大きい」「業歴が長い → 信頼できる」「IT業種 → 新商品と相性◎」）、右側に矢印で「A / B / C」のランク表示。吹き出しからランクへ矢印が伸びる構成
- **表示タイミング：** 0:15〜0:40
- **講師のセリフ（要旨）：** 「営業の優先順位って、頭の中にありますよね。それを言葉にしてAIに伝える。AIがリスト全部に当てはめてくれます。」

---

### 【スライド3：Before/After比較】

- **表示テキスト：** （左）ただのリスト →（右）戦略リスト
- **サブテキスト：** 列が2つ増えるだけで、"迷い"が消える
- **ビジュアル要素：** 左半分にシンプルな企業リスト（5行分、会社名・住所・電話番号だけ）、右半分に同じリストに「優先度」「判定理由」の2列が追加された表。優先度A行は薄い緑、B行は薄い黄色、C行は薄い灰色でハイライト
- **表示タイミング：** 2:20〜2:55
- **講師のセリフ（要旨）：** 「左がさっきまでの状態。右が今。列が2つ増えただけで、"上から順に電話すればいい"になった。もう迷わなくていいんです。」

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示

このレクチャーのテンションは**「静かな驚き → 実感」**。大げさに驚くのではなく、「こんなこともできるんだ」という発見を自然体で伝える。特に出力結果を見せる場面（1:50〜2:20）では、講師自身が結果を読んで「へぇ、なるほど」とリアクションするくらいがちょうどいい。受講者の深層心理としては「毎日なんとなくリストの上から電話していた無力感」に寄り添い、「もう迷わなくていい」という解放感を届ける。

### NGワード

| NG表現 | 言い換え |
|--------|---------|
| アルゴリズム | AIの判断のしくみ |
| スコアリング | ランク付け、点数付け |
| パラメータ | 条件、基準 |
| ロジック | 考え方、判断の流れ |
| プライオリティ | 優先度、優先順位 |
| セグメント | グループ分け |
| データドリブン | データを使った判断 |
| フィルタリング | 絞り込み（※4.1で使用済み。本レクチャーでは「ランク付け」に統一） |

### カメラ・マイク

- **顔出し：** 不要。画面共有のみ。
- **マイク：** 通常の収録用マイク。タイピング音が入らないよう、プロンプトはコピー＆ペーストで入力する。
- **画面収録：** Claudeの画面を全画面で収録。ブラウザのブックマークバーは非表示にする（個人情報が映り込まないため）。

### 編集指示

- **0:40〜1:10（データ貼り付け）：** 15社分のデータを貼り付ける操作は、2倍速に早送りする。全データが表示された状態で通常速度に戻す。
- **1:50〜2:20（Claude出力待ち）：** Claudeが表を生成している間のローディング部分は、3倍速でカット。出力が完了した瞬間から通常速度に戻す。
- **2:55〜3:25（2回目の出力待ち）：** 同様にローディング部分を3倍速でカット。
- **テロップ追加箇所：**
  - 1:10のプロンプト貼り付け時：画面下部に「※プロンプトは特典テンプレートからコピーできます」のテロップを3秒間表示
  - 3:25の注意点パート冒頭：画面上部に「💡 AIの判定は"参考意見"。最終判断はあなた」のテロップを5秒間表示

### 前後のレクチャーとの接続

- **冒頭（0:00〜0:15）：** 「前回のレクチャーで、リストをきれいにする方法をやりました」と一言だけ触れる。「重複排除」「業種フィルタ」というキーワードを出して、4.1の内容を思い出させる。
- **締め（3:45〜4:00）：** 「次のセクションでは、このランク付きリストを使って、エリア分けと訪問計画を作ります」と予告。S5への橋渡しとして、「"月曜はこのエリアの5件"みたいに1週間の予定が組める」というベネフィットを1文だけ伝える。
