# レクチャー3.2 詳細設計書

## 【失敗と対処】スキャンPDF・複雑レイアウトで崩れた時のリカバリー術

---

## ① レクチャーの位置づけ

レクチャー3.1で受講者は「テキストPDFをClaudeにアップロードするだけで、きれいな表が一発で出る」という成功体験を得ている。しかし現場のPDFはテキストベースとは限らない。本レクチャーでは、スキャンPDF（画像PDF）や段組み・複雑レイアウトのPDFで**抽出が崩れる現実**を正直に見せ、そのリカバリー方法を2パターン（OCRアプリでの前処理／列名を指定した再指示）教える。受講者が得る1つのゴールは「**PDFの読み取りが崩れても、自分で直せるという安心感を手に入れること**」である。これにより3.3（Webページからの抽出）に進む際、「PDFでもWebでも、うまくいかなければ直せばいい」という心理的な土台が完成する。また、講座全体の信条である「成功例だけ見せない」を体現する最重要レクチャーであり、ここでの正直な姿勢が受講者の信頼を決定づける。

## ④ ライブ実演の操作手順

### 実演パートA：スキャンPDFの失敗体験（0:30〜1:10）

```
【実演パート名：スキャンPDFの抽出失敗】

＜操作手順＞
1. Claudeの画面を開く（新規チャットが表示されている状態）
2. 画面左下のクリップアイコン（ファイル添付ボタン）をクリック
3. ファイル選択ダイアログから「製造業_協会名簿_スキャン.pdf」を選択し「開く」をクリック
4. PDFのプレビューが表示されたことを確認（※受講者に「これがスキャンPDFです」と見せる間を取る）
5. 事前にコピーしておいたプロンプト①をCtrl+Vで入力欄に貼り付ける：
   ────────────
   このPDFから会社名・代表者名・住所・電話番号・業種を表形式で抽出してください。
   ────────────
6. Enterキーで送信
7. Claudeの出力を待つ（5〜10秒）
8. 出力結果を画面に表示したまま、崩れている箇所（文字化け・列ずれ・空欄）をマウスカーソルで指し示す

＜講師が強調すべきポイント＞
- 「前回と全く同じやり方なのに、結果がこんなに違います」
- 「これは、あなたのやり方が悪いんじゃなくて、PDFの種類が違うからです」

＜想定トラブルと対処＞
- Claudeが意外ときれいに読み取ってしまった場合：PDFの画質をさらに落としたバージョンを予備として用意しておく。または「今回はたまたまうまくいきましたが、もっと古いスキャンだとこうなります」と別の事例画像をスライドで補足する
- Claudeが「画像のため読み取れません」とエラーを返した場合：「はい、こういうふうに"読めません"と正直に言ってくれることもあります。これはこれでわかりやすいですね」とポジティブにフォロー
```

```
- Claudeが誤読を補正しきれなかった場合：「こういう時は、間違っている箇所を指摘して"ここは○○の間違いだと思うので直して"と追加で指示すれば直ります」とフォロー
```

## ⑤ スライドの指示

```
【スライド1：PDFには2種類ある】
- 表示テキスト：「PDFは2種類ある」
- サブテキスト：左半分「テキストPDF → AIが読める ✓」／右半分「スキャンPDF → AIが読めない ✗」
- ビジュアル要素：左に文字がくっきり映ったPDFのアイコン、右にスキャナーから出てきた紙のイラスト（または写真アイコン）。矢印で「コピー機でスキャン＝ただの写真」と注釈。色分けは、左が緑系、右がオレンジ系で直感的に区別できるようにする。
- 表示タイミング：0:00〜0:30
- 講師のセリフ（要旨）：「前回うまくいったのはテキストPDFだったからです。スキャンしたPDFはAIにとっては写真と同じ。文字が写っているだけで、中身は読めません」
```

```
【スライド2：崩れた時のリカバリー2つの方法】
- 表示テキスト：「崩れても大丈夫。直し方は2つ」

- ビジュアル要素：シンプルなフローチャート。分岐を2つに絞り、矢印と吹き出しで構成。フローチャートの各ボックスは大きめの文字で、スマホでも判読可能なサイズにする。

```

```
- 表示テキスト：「指示のコツ：場所と列名を絞る」

```

## ⑥ 講師メモ（撮影時の注意点）

**トーンの指示：**
このレクチャーは「失敗を見せる」回なので、失敗が出た瞬間に慌てたり焦ったりしないこと。むしろ「ほら、こうなるんです」と**落ち着いた、少し楽しそうなトーン**で見せる。「失敗＝ダメ」ではなく「失敗＝想定内。直し方を知っていれば問題ない」という空気を作ることが最優先。受講者は「自分がやっても失敗するかも」という不安を抱えているので、講師が余裕を見せることで安心感を与える。

**NGワード：**

| 使ってはいけない表現 | 言い換え |
|---|---|
| OCR（初出時に説明なく使う） | 「写真の文字を読み取る機能」（初出時に説明した後は「OCR」も併用可） |
| テキスト抽出 | 「文字を取り出す」 |
| レイアウト解析 | 「ページの構造を読み取る」 |
| パース | 使わない（言い換え不要、この概念自体に触れない） |
| データポリシー | 「データの扱い方のルール」 |
| API | 「裏側の仕組み」 |
| ハルシネーション | 「AIが勝手に作ってしまうこと」 |

**カメラ・マイク：**
