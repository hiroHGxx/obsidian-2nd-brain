# レクチャー詳細設計書：6.1 DM教材不要！AIに"営業DMの基本構成"を教えてもらう

---

## ① レクチャーの位置づけ

本レクチャーはセクション6「刺さるDMを量産する」の導入であり、S3〜S5で整形・戦略化した営業リストを**実際の営業アクションに変換する転換点**となる。S5までで「リストは完成した。じゃあ次は何をする？」という受講者の期待に応え、「DMの書き方そのものをAIに教わる」という新しい体験を提供する。前レクチャー5.3の「訪問計画をAIに作らせた」成功体験を引き継ぎ、「AIは"作業の代行"だけでなく"知識の提供"もしてくれる」という認識のアップデートが本レクチャーの1つのゴールである。このゴールが、次の6.2「業種別DM一括生成」で実際にDMを量産する実演への土台となる。

### プロンプト全文

**プロンプト①（DMの基本構成を聞く）：**

```
あなたは法人営業のDMライティングの専門家です。

中小企業の新規開拓をしている営業マンが、メールDMを送りたいと考えています。
反応率が高い法人営業DMの「基本構成」を教えてください。

以下の項目について、それぞれ何を書くべきか、ポイントと一緒に教えてください。
1. 件名
2. 冒頭の書き出し（最初の2行）
3. 本文（提案内容の伝え方）
4. 実績・根拠
5. CTA（相手に取ってほしい行動）

※ 営業経験はあるがDMの書き方を学んだことがない人向けに、わかりやすくお願いします。
```

**プロンプト②（具体例を出させるフォロー）：**

```
ありがとうございます。
では、この構成に沿って「製造業の工場向けに、業務効率化のITサービスを提案するDM」のサンプルを1通だけ書いてみてください。
```

## ④ ライブ実演の操作手順

```
【実演パートA：Claudeに"DMの基本構成"を聞く】

＜操作手順＞
1. Chromeブラウザでclaude.aiを開く（事前にログイン済み）
2. 画面左上の「New chat」をクリックして新規チャットを開始
3. 入力欄にカーソルを合わせる
4. Ctrl+V（Mac: Cmd+V）でプロンプト①を貼り付ける
   ※ タイピングはしない。貼り付けることで時間を節約し、タイプミスも防ぐ
5. 貼り付けたプロンプトが入力欄に表示されたことを目視確認（一拍置く）
6. Enterキーを押して送信
7. Claudeが回答を生成するのを待つ（5〜15秒）
8. 回答が完了したら、画面をゆっくり上から下にスクロールし、5つの項目を順番に見せる
9. 「1. 件名」のセクションでスクロールを一旦止め、マウスカーソルで「相手のメリット」に関する記述を指し示す
10. 「5. CTA」のセクションでスクロールを一旦止め、マウスカーソルで「負担を小さく」に関する記述を指し示す

＜講師が強調すべきポイント＞
- 「10秒です。DM構成のポイントが10秒で手に入りました。」
- 「覚えなくていいんです。毎回AIに聞けばいい。自分でいつでも作れるんです。」

＜想定トラブルと対処＞
- Claudeの回答が5項目ではなく3〜4項目にまとまった場合：
  「AIが自分なりに整理してくれましたね。指定した項目と少し違っても、
  必要な内容はカバーされています。気になれば"5つに分けて"ともう一度お願いすればOKです。」
- 回答が長すぎてスクロールが多い場合：
  すべてを読み上げず、件名とCTAの2箇所だけ拾って次に進む
```

```
【実演パートB：具体的なDMサンプルを生成させる】

＜操作手順＞
1. 同じチャット内で、入力欄にカーソルを合わせる
2. Ctrl+V（Mac: Cmd+V）でプロンプト②を貼り付ける
3. 貼り付け内容を目視確認（一拍置く）
4. Enterキーを押して送信
5. Claudeが具体的なDMサンプルを生成するのを待つ（5〜15秒）
6. 回答が完了したら、画面をゆっくりスクロールしてDM全文を見せる
7. 件名部分でスクロールを止め、マウスカーソルで指し示す
8. CTA部分（末尾の行動喚起）でスクロールを止め、マウスカーソルで指し示す

＜講師が強調すべきポイント＞
- 「このまま使えるレベルですよね。あとは自分の会社名と商品に書き換えるだけ。」
- 「しかもこれ、"製造業向け"って指定したから製造業に刺さる言葉遣いになっている。業種を変えれば、それに合わせた言い回しにしてくれます。次のレクチャーでそれをやります。」

＜想定トラブルと対処＞
- 生成されたDMが英語混じりになった場合：
  「たまに英語が混ざることがあります。そのときは"すべて日本語で書き直して"と追加で指示すれば直ります。」
- DMの内容が自分の業界と違う場合（受講者の心理を先回り）：
  「"製造業向け"で例を出しましたが、あなたの業界名に変えれば、その業界向けの表現に変わります。次のレクチャーで業種別に一気に作る方法をやりますので、お楽しみに。」
```

## ⑥ 講師メモ（撮影時の注意点）

**トーンの指示：**
このレクチャーは「驚き＋安心」がテーマ。「えっ、こんなことまで教えてくれるの？」という驚きと、「自分で一から勉強しなくていいんだ」という安心感を届ける。語り口は"先輩が隣で教えてくれる"テンション。興奮しすぎず、でも淡々としすぎない。Claudeの回答を見たときに「おお、いいですね」と自然に感嘆する程度のリアクションが理想。

**NGワード：**

| NGワード | 言い換え |
|---------|---------|
| プロンプトエンジニアリング | 指示の書き方、AIへのお願いの仕方 |
| CTA | 「相手に取ってほしい行動」（※スライドでは"CTA"を使うが、口頭では必ず言い換える） |
| コンバージョン | 反応、返信 |
| テンプレート（文脈なしで使う場合） | ひな形、たたき台 |
| LLM、大規模言語モデル | AI |
| 生成AI | AI（シンプルに） |

**カメラ・マイク：**
画面共有のみ。顔出し不要（画面の情報量が多いため、ワイプも不要）。マイクは単一指向性のヘッドセットまたはコンデンサーマイク推奨。キーボードのタイピング音が入らないよう注意（コピペ操作のため最小限だが、Ctrl+Vの音にも配慮）。

**編集指示：**
- 0:45〜1:10のプロンプト貼り付け部分：貼り付け後、プロンプト全文が見える状態で1.5秒静止するようカット編集
