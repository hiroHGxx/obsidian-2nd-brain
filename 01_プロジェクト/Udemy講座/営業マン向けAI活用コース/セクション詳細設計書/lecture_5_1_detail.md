# レクチャー5.1 詳細設計書

## Claudeで"ざっくりエリア分け" ― 市区町村グループ分けを一瞬で

---

## ① レクチャーの位置づけ

S4（セクション4）で受講者は営業リストの重複排除・業種フィルタ・優先度ランク付けを習得し、「ただの一覧」を「優先順位付きの戦略リスト」に仕上げた。本レクチャーはS5（エリア分け＋訪問計画）の入口であり、その戦略リストを**地理的に分類する**という新しい切り口を加える最初の一歩である。受講者が得る「1つのゴール」は、**営業リストをClaudeに渡して市区町村レベルのエリアグループに自動仕分けできるようになること**。同時に「Claudeは正確な距離計算が苦手」という限界を正直に伝え、次のレクチャー5.2（Googleマイマップ連携）で地図上に可視化して補完する流れへ橋渡しする。

### エリア分け指示プロンプト（全文）

```
あなたは愛知県の地理に詳しい営業アシスタントです。

以下の営業リストを、私の営業所（名古屋市中区丸の内2-1-1）を起点にして、
訪問しやすいエリアごとにグループ分けしてください。

【グループ分けのルール】
・市区町村の位置関係をもとに、3〜5グループに分ける
・各グループに「名古屋市内」「尾張エリア」などのわかりやすいエリア名をつける
・グループごとに会社名・住所・優先度を表形式で出力する
・各グループの先頭に、そのエリアの特徴（おおよその方角・距離感）を1行で添える
```

## ④ ライブ実演の操作手順

```
【実演パート名：営業リストのエリア別グループ分け】

＜操作手順＞
1. Claudeの画面を開く（Google Chromeで https://claude.ai にアクセス済みの状態）
2. 画面左上の「＋」ボタン（新しいチャット）をクリックして新規チャットを開く
3. 画面下部の入力欄の左にあるクリップアイコン（ファイル添付）をクリック
4. ファイル選択ダイアログが開く → デスクトップに保存してある
   「愛知県_営業リスト_優先度付き.csv」をダブルクリックして選択
5. 入力欄の上にファイル名「愛知県_営業リスト_優先度付き.csv」が
   表示されたことを確認（※ここで一拍置いて受講者に画面を見せる）
6. テキストエディタ（メモ帳やVSCode等）に事前コピーしておいた
   プロンプト全文を選択してコピー（Ctrl+A → Ctrl+C）
7. Claudeの入力欄をクリックしてカーソルを合わせる
8. プロンプトを貼り付け（Ctrl+V）
9. 貼り付けたプロンプトの内容を指差しながら3つのポイントを口頭で説明
   （営業所住所の起点指定、市区町村ベース、表形式出力）
10. Enterキーで送信
11. Claudeが出力を生成するのを待つ（5〜10秒）
12. 出力結果をゆっくりスクロールしながら表示する
    - まず「名古屋市内」グループが表示される → 指差して社数を確認
    - 次に「尾張エリア」グループ → 指差して社数を確認
    - 最後に「三河エリア」グループ → 指差して社数を確認
13. 各グループの先頭に添えられたエリア特徴コメントを指差して見せる

＜講師が強調すべきポイント＞
- 「20社のリストが、たった1回の指示で3つのエリアに仕分けされました」
- 「これ、手作業でやったら地図を見ながら15分はかかりますよね」
- 「もちろん完璧ではありません。皆さんの土地勘で微調整するのが前提です」

＜想定トラブルと対処＞
- Claudeが4グループ以上に分けた場合：
  「3〜5と指示しているので範囲内です。もし3つにまとめたければ
  "3グループにまとめ直して"と追加指示すれば修正してくれます」とフォロー
- 特定の市区町村が直感と違うグループに入った場合：
  「こういう"ん？"ということもあります。Claudeは地図を見ているわけではなく、
  市区町村の名前と地域の知識でざっくり判断しています。
  だからこそ、次のレクチャーで地図上に可視化して確認するんです」と説明
- Claudeの応答が遅い場合：
  「ファイルを読み込みながら分類しているので少し時間がかかることもあります。
  手で地図を見ながら分けるよりは遥かに速いですね」とフォロー
```

## ⑥ 講師メモ（撮影時の注意点）

- **トーンの指示：** 「お、これ便利じゃん」と思ってもらえる軽快なテンション。驚かせるのではなく、「あ、これなら自分でもできそう」という安心感を優先する。S5の入口なので、S4の達成感を引き継ぎつつ「次のステップに進もう」というワクワク感を出す
- **NGワード：**
  - NG「ジオコーディング」→ 言い換え「住所から場所を割り出すこと」
  - NG「クラスタリング」→ 言い換え「グループ分け」
  - NG「API」→ 言い換え「裏側の仕組み」
  - NG「緯度経度」→ 言い換え「地図上の位置情報」
  - NG「パラメータ」→ 言い換え「条件」
- **カメラ・マイク：** 顔出しなし、画面共有のみ。マイクはピンマイクまたはヘッドセットマイクで、タイピング音が入らないよう注意（プロンプトは貼り付けで対応するため基本的にタイピング音は出ない）
- **編集指示：**
  - Claudeの出力待ち（1:25〜1:45）で10秒以上かかった場合は5秒程度にカット編集する。カット時に画面右下に「※出力待ち：約10秒」のテロップを表示
  - 出力結果のスクロール確認パート（1:45〜2:10）では、各グループ名が画面に映った瞬間にグループ名をハイライトするテロップを追加（「名古屋市内：8社」「尾張エリア：6社」「三河エリア：6社」）
- **前後のレクチャーとの接続：**
  - 冒頭：S4への言及は不要。スライド1の「地図とにらめっこ」で受講者の日常に直接入る
  - 末尾：「次のレクチャーでは、今Claudeが作ってくれたこのリストをCSVに出力して、Googleマイマップにインポートします。地図にピンが立つ瞬間、結構テンション上がりますよ」と5.2を予告する
