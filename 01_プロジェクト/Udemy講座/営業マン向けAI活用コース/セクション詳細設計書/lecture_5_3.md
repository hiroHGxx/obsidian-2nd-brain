# レクチャー5.3 詳細設計書

## "1日5件、エリアごとに"の訪問計画をAIに作らせる

---

## ① レクチャーの位置づけ

レクチャー5.2で受講者はClaudeが出力したCSVをGoogleマイマップにインポートし、訪問先が地図上にピンとして可視化される体験を得ている。「地図で見える化できた」という達成感がある一方、「で、どの順番で回ればいいの？」という次の問いが自然に生まれる。本レクチャーはその問いに直接応え、**エリア分けされたリストから「月曜はAエリア5件、火曜はBエリア5件」の週間訪問計画をClaudeに自動生成させる**ことが1つのゴールである。同時に、Claudeは正確な距離計算や最短ルート算出が苦手であるという限界を正直に示し、「ざっくり計画はClaude、正確なルート確認はGoogleマップ」という使い分けの原則を定着させる。本レクチャーでセクション5の「エリア分け＋訪問計画」パートが完結し、次のS6ではこの訪問計画に紐づく企業へのDM一括生成へと展開する。訪問計画という"行動の設計図"が手元にある状態でDM作成に入ることで、「誰に・いつ・何を送るか」が一気通貫でつながる流れを作る。

## ④ ライブ実演の操作手順

```
【実演パート名：週間訪問計画の生成とGoogleマップでの確認】

＜操作手順＞

[Phase 1：Claudeでの訪問計画生成]
1. Claudeの画面を開く（新規チャット、またはレクチャー5.1〜5.2で使用したチャットの続き）
2. 入力欄をクリックする
3. 事前準備した15件のエリア分けリスト（表形式テキスト）をテキストエディタからCtrl+C → Claudeの入力欄にCtrl+Vで貼り付け
4. 改行を1行入れる
5. 事前準備したプロンプト全文をテキストエディタからCtrl+C → Claudeの入力欄にCtrl+Vで貼り付け
6. 貼り付けたプロンプトの「1日の訪問件数は最大5件」「優先度Aの企業を週の前半に配置」の部分を指差して強調
7. Enterキーで送信
8. Claudeが出力を生成するのを待つ（5〜10秒）
9. 出力が完了したら、画面をゆっくり下にスクロールし、月曜〜金曜の計画全体を見せる
10. 月曜の表を指差し、優先度A企業が含まれていることを確認する
11. 水曜〜木曜の表を指差し、エリアが「尾張エリア」でまとまっていることを確認する

[Phase 2：Googleマップでのルート確認]
12. ブラウザ上部のタブからGoogleマップのタブをクリックして切り替え
13. Claudeの出力に戻り、月曜1件目の住所「愛知県名古屋市中区丸の内2-5-10」をドラッグで選択 → Ctrl+C
14. Googleマップのタブに戻り、検索窓をクリック → Ctrl+V → Enterで検索
15. 地図上にピンが立ったことを確認
16. 検索窓の下に表示される「ルート」ボタンをクリック
17. 出発地に「愛知県名古屋市中区栄3-1-1」と入力（営業所の住所）
18. 「目的地を追加」をクリックし、月曜2件目の住所「愛知県名古屋市中区栄4-8-3」をペースト
19. 経路と所要時間が表示されたことを画面に映す
20. 「この所要時間を見ながら、訪問順を微調整すればOKです」とコメント

＜講師が強調すべきポイント＞
- 「Claudeに渡す条件は4つだけ。件数・エリアまとめ・優先度・出発地点。」
- 「自分で並べ替えたら何分かかりますか？Claudeなら10秒です。」
- 「ざっくり計画はClaude、正確なルートはGoogleマップ。この使い分けが大事。」

＜想定トラブルと対処＞
- Claudeが1日に6件以上を割り当てた場合：
  → 「"最大5件と書いたのに6件入ってますね。こういう時は『月曜が6件になっています。5件以内に修正してください』と追加で指示すればすぐ直ります"」と実演
- Claudeが優先度を無視してランダムに配置した場合：
  → 「"優先度Aが木曜に入ってますね。『優先度Aを月〜水に集めてください』ともう一度伝えましょう"」とフォロー
- Googleマップで住所が正確にヒットしない場合：
  → 「"架空の住所なので少しずれることもありますが、実際の住所なら正確にピンが立ちます"」と補足
```

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示
- テンションは「先輩が後輩に裏ワザを教える」くらいのカジュアルさ。感動を押し付けず、「便利でしょ？」という軽さで進める
- 操作スピードは速すぎず、受講者が「自分でもできそう」と感じるペースを維持する
- Googleマップ連携パートでは、Claudeの限界を正直に伝えるトーンを意識する。「Claudeはすごい！」ではなく「ここはClaudeの苦手分野なので、別のツールを使いましょう」というフラットな姿勢

### NGワード

| NGワード | 言い換え |
|---------|---------|
| アルゴリズム | 「AIの考え方」 |
| 最適化 | 「一番いい順番を考える」 |
| パラメータ | 「条件」 |
| ルーティング | 「ルートを決める」「回る順番を決める」 |
| API | 使用しない（本レクチャーでは不要） |
| プロンプトエンジニアリング | 「指示の出し方」 |

### カメラ・マイク
- 画面共有メイン。講師の顔はワイプ（画面右下・小サイズ）で表示
- 冒頭（0:00〜0:15）と末尾（2:50〜3:00）のみカメラ目線で話す
- ライブ実演中はワイプを維持しつつ、視線は画面操作に向ける

### 編集指示
- 0:30〜1:00のプロンプト貼り付けシーン：テキストエディタからのコピーは実際の操作を見せるが、もたつく場合はカット編集で短縮してよい
- 1:05〜1:20のClaude出力待ち：実際の生成時間が10秒以上かかる場合は早送り編集（×2速）で短縮。画面右上に「×2速」のテロップを入れる
- 2:10〜2:35のスライド表示：スライドへの切り替え・戻しのトランジションはカット（フェードなし）でテンポよく
- テロップ挿入箇所：
  - 1:00 送信の瞬間に「送信！」のテロップ（ポップアップ・0.5秒表示）
  - 1:20〜1:45 Claudeの出力結果を映している間、画面下部に「ポイント：優先度Aが月〜水に集中」のテロップ
  - 2:10 スライド切り替え時に「★ ここだけ覚えて！」のテロップ

### 前後のレクチャーとの接続
- **冒頭（前レクチャーとの接続）：** 5.2で「マイマップにピンが立った」ことを前提として触れ、「でもどの順番で回るか」という次の課題を提起して本題に入る。5.1のエリア分けリストをそのまま使い回すことで、セクション全体の一貫性を示す
- **末尾（次レクチャーへの橋渡し）：** 「リスト抽出→エリア分け→訪問計画、ここまで全部つながりました」とS3〜S5の流れを一言で振り返り、「次は、この訪問先に送るDMをAIに一括で作ります」とS6への期待を持たせて終わる。S5の完結感とS6への接続を同時に行う
