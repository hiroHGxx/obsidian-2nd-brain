## ① レクチャーの位置づけ

本レクチャーはセクション2「AIを安心して使い始める」の冒頭であり、受講者が**初めてAIツールの画面に触れる瞬間**を担う。S1で「手作業がこう変わる」という未来像を見せて期待感を高めた直後のパートであり、「じゃあ実際どうやって始めるの？」という受講者の次の疑問にダイレクトに応える役割を果たす。

- **前のレクチャー（1.2）から引き継ぐもの：** Before/Afterのデモで高まった「自分もやってみたい」というモチベーション。ここではその熱が冷めないうちに、最短手順でアカウント作成〜最初のファイルアップロードまで完了させる。
- **このレクチャーのゴール：** 受講者がClaudeのアカウントを作成し、PDFファイルをアップロードして最初の指示を送信できる状態になること（ChatGPTも同様の手順を把握している状態）。
- **次のレクチャー（2.2）への橋渡し：** アカウントを作れた受講者が次に気にする「無料のままでどこまでできるの？」「有料にしないとダメ？」という疑問を予告し、2.2へ自然につなげる。

## ④ ライブ実演の操作手順

```
【実演パート名：PDFアップロードと最初の指示送信】

＜前提＞
- Claudeにログイン済みの状態でスタート
- 新規チャット画面が表示されている
- テスト用PDF「愛知県_商工会議所_企業一覧.pdf」はデスクトップに配置済み
- プロンプト文はメモ帳アプリで開いた状態でタスクバーに待機

＜操作手順＞
1. Claudeの新規チャット画面を表示する（画面中央に「How can I help you today?」等の初期メッセージが見えている状態）
2. 画面下部の入力欄の左側にあるクリップマーク（📎アイコン）をクリック
3. ファイル選択ダイアログが開く → デスクトップの「愛知県_商工会議所_企業一覧.pdf」をダブルクリックして選択
4. 入力欄の上にPDFのファイル名「愛知県_商工会議所_企業一覧.pdf」が表示されたことを画面上で確認する（※ここで1〜2秒静止し、受講者に「ファイルが読み込まれた」ことを視認させる）
5. タスクバーからメモ帳アプリに切り替え、以下のプロンプト全文を選択してコピー（Ctrl+A → Ctrl+C）：
   ────────────
   このPDFに載っている企業の一覧を、表にまとめてください。
   ────────────
6. Claudeの画面に戻り、入力欄をクリックしてカーソルを置く
7. Ctrl+Vでプロンプトを貼り付け
8. 入力欄にプロンプトが表示されたことを確認（※1秒静止）
9. Enterキーを押して送信
10. Claudeが処理中の表示（●●●の点滅アニメーション）になる → 「今読み取っています」とナレーション
11. 表が出力されたら、画面をスクロールせず、表全体が見える状態で2〜3秒静止
12. 出力された表の「会社名」列を指差し → 「住所」列を指差し → 「電話番号」列を指差し → 「業種」列を指差し、各列が正しく抽出されていることを視覚的に示す

＜講師が強調すべきポイント＞
- 「ファイルを選んで、ひとこと指示を出すだけ。これだけです」
- 「細かい設定とか、難しい操作は何もありません」
- 「手で打ち込んだら何分かかるか…考えたくもないですよね」

＜想定トラブルと対処＞
- PDFアップロード後にファイル名が表示されない場合：「もう一度クリップマークから選び直してください。たまにありますが、焦らなくて大丈夫です」
- Claudeの応答が10秒以上かかる場合：「混み合っている時間帯だと少し待つこともあります。手入力の時間に比べれば誤差ですね」
- 表の一部が崩れて出力された場合：「こういう崩れは実際にあります。直し方はセクション3で詳しくやりますので、今は"だいたい読み取れた"で大丈夫です」
```

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示
- **テンション：** 明るく、軽快に。「難しいことをこれから教えます」ではなく「びっくりするくらい簡単です」という空気を作る。友人に「これ使ってみなよ、超カンタンだから」と勧めるような語り口。
- **スピード：** 操作説明中はゆっくり（受講者が一緒にやれるペース）。セリフ部分は普通のテンポ。間を恐れないこと。

### NGワード

| NG表現 | 言い換え |
|--------|---------|
| アカウント登録 | アカウントを作る |
| サインアップ | 新規登録、始める |
| UI（ユーアイ） | 画面の見た目 |
| インターフェース | 画面 |
| アップロード | ファイルを渡す、ファイルを選ぶ |
| プロンプト（初出時） | AIへの指示、指示文 |
| テキスト入力欄 | 文字を打つところ |
| レスポンス | AIの回答、返ってきた答え |
| 添付 | ファイルを渡す |

※「アップロード」「プロンプト」は画面上のボタン名や一般的な呼称として使う場面では可。ただし初出時には必ず「ファイルを渡す、いわゆるアップロードですね」のように日常語を先に出してから補足する。

### カメラ・マイク
- **顔出し：** 不要。全編画面共有＋ナレーションで構成。
- **マイク：** ナレーション収録。静かな環境で、口元から拳1つ分の距離を保って収録。
- **画面収録ソフト：** OBS Studio推奨。解像度1920×1080、フレームレート30fps。

### 編集指示
- **0:15〜0:50（アカウント作成パート）：** 事前にアカウント作成を画面収録し、メールアドレス入力・パスワード設定・メール認証の待ち時間をカット編集で短縮。実際の操作時間3〜5分を約35秒に凝縮する。カット箇所にはクロスフェードトランジション（0.3秒）を入れ、飛んだ感を出さない。
- **1:20〜2:10（ライブ実演パート）：** ここはカットせずリアルタイム収録。AI の応答待ち時間も含めてそのまま見せることで、受講者が実際の体験と同じ感覚を持てるようにする。
- **テロップ追加箇所：**
  - 0:15 画面上部に「claude.ai にアクセス」のテロップ（白背景・黒文字・角丸帯）
  - 0:30 「"Googleで続行"をクリック」のテロップ
  - 1:25 「📎 クリップマークをクリック」のテロップ
  - 1:40 画面下部に指示文のテロップ表示：`このPDFに載っている企業の一覧を、表にまとめてください。`
  - 2:15 画面上部に「chatgpt.com にアクセス」のテロップ

### 前後のレクチャーとの接続
- **冒頭（0:00）：** 前のレクチャーに直接触れず、「さっそくAIを使い始めましょう」とスムーズに導入。S1で見た未来像への期待感を暗黙的に引き継ぐ。
- **末尾（2:40〜3:00）：** 「次のレクチャーでは、"無料のままでどこまでできるの？"という疑問にお答えします。結論から言うと、まずは無料で十分です」と予告。結論を先に言うことで、次も観たくなる構成にする。
