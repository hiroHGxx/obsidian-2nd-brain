# レクチャー 3.4 詳細設計書

## プロンプトの"型"を理解する ― なぜこう書くと精度が上がるのか

**レクチャーID:** 3.4  
**レクチャー時間:** 3分  
**所属セクション:** S3: PDFとWebから企業データを一発抽出する ― 手入力ゼロの衝撃

---

## ① レクチャーの位置づけ

レクチャー3.1〜3.3で受講者は「PDFからの表データ抽出」「スキャンPDFの崩れ対処」「Webページからのコピペ整形」を体験済みである。その過程で複数のプロンプトを"言われた通りに"コピペして使ってきたが、「なぜその書き方をすると精度が上がるのか」の理解がないため、自力で書き換えることができない。本レクチャーでは、S3で実際に使ったプロンプトを"分解"し、3つの構成要素（役割指定・具体的指示・出力形式指定）がそれぞれ精度にどう影響するかを視覚的に解説する。これにより受講者は「型を知っているからテンプレートを自分の業務に合わせてカスタマイズできる」状態になる。S4以降では新しい種類のプロンプト（重複排除・優先度ランク付け等）が登場するため、その橋渡しとして「型さえ押さえれば、何を指示するときも同じフレームで書ける」という自信を本レクチャーで持たせる。

## ⑥ 講師メモ（撮影時の注意点）

**トーンの指示：**  
S3の実践パートが続いた後の"種明かし"レクチャーであるため、テンションは落ち着いたトーンで「実はあのプロンプト、こういう仕組みだったんですよ」と裏側を見せるガイド役の語り口にする。「勉強」感を出さず、「知っておくと得する豆知識」くらいの軽さを保つ。

**NGワード（使ってはいけない表現 → 言い換え）：**

| NGワード | 言い換え |
|---------|---------|
| プロンプトエンジニアリング | プロンプトの書き方の工夫 |
| トークン | （言及しない。必要なら「AIが処理する文字の量」） |
| パラメータ | （言及しない） |
| システムプロンプト | （言及しない） |
| ペルソナ設定 | 役割を与える |
| 構造化データ | 表形式のデータ |
| 出力フォーマット | 出力の形・見た目 |

**カメラ・マイク：**  
画面共有のみ（スライド表示）。講師の顔出しは不要。ナレーションはマイク収録。スライドの切り替えタイミングに合わせてナレーションを当てる。

**編集指示：**

| 箇所 | 編集内容 |
|------|---------|
| 0:40〜1:30（スライド3） | 3つの色分けブロックを①②③の順にアニメーションで順次表示（一度に出さない）。ナレーションに合わせて1ブロックずつフェードインさせる。 |
| 1:30〜2:15（スライド4） | 3行をナレーションに合わせて1行ずつ表示するアニメーション。 |
| 全体 | 各スライド切り替え時にワイプまたはフェードのトランジションを入れる（派手な演出は不要）。 |
| 全体 | スライド上のプロンプト文が読みやすいよう、フォントサイズは最低20pt以上を確保。 |

**前後のレクチャーとの接続：**

- **冒頭（前レクチャーからの接続）：** 「ここまでのレクチャーで、PDFやWebページからデータを抜き出すのを体験してもらいました」と、3.1〜3.3の実践体験を振り返る一言を入れる。個別のレクチャー名は挙げず、"体験"という言葉で束ねる。
- **末尾（次セクションへの橋渡し）：** 「次のセクションでは、この抜き出したデータを"ただの一覧"から"戦略的な営業リスト"に仕上げていきます」と、S4のテーマ（重複排除・優先度ランク付け）を"営業リスト"というキーワードで予告する。S3を締めくくるレクチャーなので、セクション間の移行を意識した語り口にする。
