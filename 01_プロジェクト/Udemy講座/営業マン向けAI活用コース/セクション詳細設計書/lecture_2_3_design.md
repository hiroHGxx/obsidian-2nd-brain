# レクチャー詳細設計書：2.3「会社のデータ、AIに取られない？」― 3分でわかる安心設定

---

## ① レクチャーの位置づけ

前のレクチャー2.2「無料プランvs有料プラン」で「お金の不安」を解消した受講者に対し、本レクチャーでは次に大きな壁となる「セキュリティの不安」を正面から取り除く。営業マンが生成AIの利用をためらう最大の理由は「顧客情報や社内データがAIの学習に使われるのでは？」という恐怖であり、この恐怖が残ったままではS3以降の実践パートで手が止まる。本レクチャーのゴールは、**「Claudeはそもそもデータを学習に使わない。ChatGPTも設定1つで学習除外できる」という2つの事実を理解し、安心してAIにデータを渡せる状態になること**である。次のレクチャー2.4「社内でAI利用許可を勝ち取る」では、本レクチャーで得た知識を武器に上司や情シスを説得する方法を教えるため、ここで「自分自身が納得できている」状態を作ることが橋渡しとなる。

## ④ ライブ実演の操作手順（該当する場合のみ）

本レクチャーにライブ実演パートはなし。代わりに**画面収録（編集済み）**パートがあるため、その収録手順を以下に記載する。

```
【画面収録パート名：ChatGPTの学習除外設定】

＜事前収録の操作手順＞
1. ブラウザでChatGPT（https://chatgpt.com）を開く（ログイン済みの状態）
2. 画面右上のプロフィールアイコン（丸いアイコン）をクリック
3. ドロップダウンメニューから「設定」をクリック
4. 左側メニューから「データコントロール」をクリック
5. 「すべての人のためにモデルを改善する」のトグルスイッチを確認する（※ここで一拍置く。編集時に赤枠とテロップを追加する箇所）
6. トグルスイッチをクリックしてオフにする（青→グレーに変化）
7. 確認ダイアログが出た場合は「オフにする」をクリック
8. トグルがグレー（オフ）になったことを画面に映して2秒静止

＜編集指示＞
- 手順2〜3の間の待ち時間はカットし、スムーズにつなげる
- 手順5の「トグルスイッチ」の箇所に赤枠と「ここをオフに！」のテロップを追加
- 手順6のトグル切替の瞬間をスロー再生（0.7倍速）にして視認性を上げる
- 全体の収録時間を編集で約30秒に圧縮する
- 画面収録中の講師ナレーション：
  「ChatGPTを開いたら、右上のアイコンから『設定』に入ります。
   次に『データコントロール』を開いて…この『モデルを改善する』のスイッチ、
   これをオフにしてください。はい、これだけです。
   この設定は1回やれば、ずっと有効です。」

＜想定トラブルと対処＞
- ChatGPTのUI変更により設定画面の項目名や配置が変わる可能性がある：
  撮影前に最新のUIを確認し、項目名が変わっていた場合はナレーションとテロップを修正する。
  講師メモとして「この画面は2025年○月時点のものです。もし画面が変わっていたら、
  "設定"の中から"データ"や"プライバシー"という言葉を探してください」とナレーションに追加する
- トグルがすでにオフになっている場合：
  収録前にいったんオンにしてから、録画中にオフにする操作を見せる

＜注意事項＞
- 画面収録時にブラウザの他のタブやブックマークバーが映り込まないよう、
  収録前にタブを1つだけにし、ブックマークバーは非表示にする
- 通知ポップアップが出ないよう、「おやすみモード」をオンにしておく
```

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示
- **安心させる語り口**を最優先。受講者は「怒られるかも」「情報漏洩したらどうしよう」という不安を抱えている。先生が教えるトーンではなく、先輩が「大丈夫だよ」と肩をたたくトーンで話す
- 声のスピードはやや遅め。セキュリティの話は早口だと不安が増す
- 「結論から言います」で冒頭に安心感を渡し、そのあとに理由を説明する構成を崩さない

### NGワード

| NGワード | 言い換え |
|---------|---------|
| データポリシー | データの扱い方のルール |
| API | （本レクチャーでは言及不要。触れない） |
| オプトアウト | 学習に使わない設定にする |
| プライバシーポリシー | 会社の公式なルール |
| トレーニングデータ | AIの学習材料 |
| エンドポイント | （本レクチャーでは言及不要。触れない） |
| コンプライアンス | 会社のルール、決まりごと |
| トグル | スイッチ、切り替えボタン |

### カメラ・マイク
- 冒頭0:00〜0:05のみ顔出し（ワイプ小窓）で信頼感を出し、以降は画面共有のみに切り替え
- マイクはピンマイクまたはヘッドセット推奨（部屋の反響を拾わないため）
- 画面収録パートはマイクをミュートにして収録し、あとからナレーションを被せる（操作音が入らないようにするため）

### 編集指示
- ChatGPTの設定変更の画面収録は、操作の待ち時間をカットして約30秒に圧縮する
- 手順5のトグルスイッチの箇所に赤枠＋「ここをオフに！」テロップをオーバーレイ
- スライド2でChatGPT側の「注意マーク → チェックマーク」の変化は、画面収録パートの直後に切り替えることで「設定したから安全になった」を視覚的に示す
- レクチャー全体の無音部分（0.5秒以上の間）は自動カットで詰める

### 前後のレクチャーとの接続
- **冒頭（前のレクチャーからの接続）：** 前のレクチャー2.2で「まず無料で始めてOK」と伝えている。本レクチャーの冒頭では「お金の心配はなくなりましたね。次に気になるのは"データの安全性"ですよね」と、不安解消の流れを自然につなげる。ただしタイムライン上は時間節約のため省略し、スライド1のタイトルで暗示する形でもよい
- **末尾（次のレクチャーへの橋渡し）：** 「自分自身は納得できた。でも上司や情シスにどう説明すればいいかわからない」という次の壁を明示し、「次のレクチャーでメールのテンプレートをそのまま渡します」と具体的な価値を予告して視聴継続を促す
