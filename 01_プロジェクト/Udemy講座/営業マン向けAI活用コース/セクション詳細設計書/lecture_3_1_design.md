# レクチャー3.1 詳細設計書

## 【成功体験】テキストPDFを表データに変換 ― アップロード1回で完了

**レクチャーID:** 3.1
**レクチャー時間:** 4分（240秒）
**セクション:** S3: PDFとWebから企業データを一発抽出する ― 手入力ゼロの衝撃

---

## ① レクチャーの位置づけ

このレクチャーは、セクションS3「PDFとWebから企業データを一発抽出する」の冒頭であり、**コース全体で最初の"手を動かす"実践パート**である。S2でアカウント作成・安心設定・プロンプトの3つの型を学び終えた受講者が、初めて「自分の業務に直結する作業」をAIで体験する瞬間にあたる。前のレクチャー（2.5）で仕込んだ「①役割を与える ②具体的に指示する ③出力形式を指定する」の3つの型を、ここで実際に使って成功体験を得ることが最大の目的である。

- **前のレクチャー（2.5）から引き継ぐもの：** プロンプトの3つの型（役割・具体指示・出力形式）の知識。「この型を使えばAIに伝わる」という理解。
- **このレクチャーのゴール：** テキストベースのPDFをClaudeにアップロードし、日本語の指示だけで企業データを表形式に変換できる、という**成功体験を1回完了させる**こと。受講者の心に「え、これだけ？自分にもできるじゃん」という感覚を刻む。
- **次のレクチャー（3.2）への橋渡し：** 「テキストPDFならこんなに簡単。でも、スキャンPDFや複雑なレイアウトだとどうなるか？」という問いを残し、失敗パターンへの興味を引く。

## ④ ライブ実演の操作手順

### 【実演パート名：PDFアップロードと表データ抽出】

#### ＜操作手順＞

1. Claudeの画面を開く（Google Chromeで `claude.ai` にアクセス済み。新規チャット画面が表示されている状態）
2. 画面下部のメッセージ入力欄の左にある**クリップ型の「ファイルを添付」アイコン**をクリック
3. ファイル選択ダイアログが開く → デスクトップ上の`愛知県_商工会議所_会員企業一覧.pdf`を選択 →「開く」をクリック
4. 入力欄の上にPDFファイル名（`愛知県_商工会議所_会員企業一覧.pdf`）がサムネイルで表示されたことを確認する（※ここで1〜2秒間を置き、受講者に「ファイルがアップロードされた」と認識させる）
5. メッセージ入力欄をクリックし、事前にクリップボードにコピーしておいた以下のプロンプトを **Ctrl+V（Mac: Cmd+V）で貼り付ける**（タイピングはしない。時間短縮と入力ミス防止のため）：

```
このPDFは愛知県の企業一覧です。以下の情報を表形式で抽出してください。

【抽出する列】
- 会社名
- 代表者名
- 住所
- 電話番号
- 業種

表はMarkdown形式で出力してください。
```

6. 貼り付けた文面を画面に2秒ほど映し、受講者に内容を読ませる時間を取る
7. **Enterキー**（またはSendボタン）で送信する。同時にタイマーを開始する
8. Claudeが応答を生成する間（3〜10秒）、画面をそのまま映し続ける。沈黙にせず、セリフで間をつなぐ
9. Claudeの出力（表）が画面に表示されたら、タイマーを停止する
10. 出力された表をゆっくり**下方向にスクロール**して全体を見せる。会社名の列、住所の列、電話番号の列を順に指さし（マウスカーソルを合わせ）ながら「合ってますね」と確認する
11. Claudeの出力テーブルの右上に表示される**「Copy」ボタン**をクリック（※Claudeのテーブル出力にはコピー機能が付いている）
12. タスクバーからExcelのウィンドウに切り替える（Alt+Tab、またはタスクバーのExcelアイコンをクリック）
13. Excel上の**A1セル**を右クリック →「貼り付けのオプション」から**「テキストとして貼り付け」**を選択（または Ctrl+V で貼り付け）
14. 表がExcelのセルに反映されたことを確認する
15. 列幅が狭い列があれば、列ヘッダーの境界をダブルクリックして自動調整する（A列〜E列）
16. 完成したExcel上のリストを2〜3秒映して見せる

#### ＜講師が強調すべきポイント＞

- 「PDFをアップロードして、指示を1回送っただけです。たったこれだけ。」
- 「タイマー見てください。約40秒。手で打ったら15分以上かかる作業です。」
- 「特別なソフトも、Excel関数も、何もいりません。」

#### ＜想定トラブルと対処＞

| トラブル | 対処法（講師のセリフ） |
|---------|------------------|
| Claudeの応答が10秒以上かかる | 「少し時間がかかることもあります。でも、手で打つよりは何十倍も速いので、コーヒーでも飲みながら待ちましょう」 |
| 表の一部の列が空欄になっている | 「あ、ここ1つ空欄になってますね。こういうのは元のPDFに書かれてなかった場合に起きます。元データを確認してみましょう」（PDFに切り替えて確認する動作を見せる） |
| Excelへの貼り付けで1セルにまとまってしまう | 「もし1つのセルに全部入っちゃった場合は、"タブ区切りで出力して"と指示を変えてみてください。そうするとExcelの列にきれいに分かれます」 |
| Claudeの出力がMarkdownのテキストのまま表にならない場合 | 「表の見た目がテキストのままの場合は、"表形式"という指示が効いてます。コピーしてExcelに貼れば表になるので問題ありません」 |

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示

- **テンション：** 明るく、テンポよく。「すごいでしょ？」ではなく「簡単でしょ？」のニュアンス。受講者を感心させるのではなく、「自分にもできそう」と思わせる語り口を意識する
- **共感の軸：** 受講者の「毎回PDFを見ながらExcelに手打ちしている無力感」に寄り添う。冒頭で「見覚えありますよね？」と問いかけ、一体感をつくる
- **結果が出た瞬間の演出：** タイマーが止まった瞬間に少しだけ間を置き、「はい、出ました」と落ち着いたトーンで伝える。大げさなリアクションは不要。数字（40秒 vs 15分）で語らせる

### NGワード

| NGワード | 言い換え |
|---------|---------|
| API | （このレクチャーでは言及不要） |
| Markdown | 「表の形」「きれいな表」（プロンプト内では使用するが、口頭では言い換える） |
| テキストベースPDF | 「文字が選択できるPDF」「コピペできるタイプのPDF」 |

| データ抽出 | 「情報を抜き出す」「表にまとめる」 |
| プレーンテキスト | 「普通の文字データ」 |

### カメラ・マイク

- **顔出し：** 冒頭スライド（0:00〜0:15）はワイプ（小窓）で顔出し。ライブ実演中（0:15〜3:45）は画面共有のみ（顔の小窓は非表示にし、画面を最大限広く使う）。クロージングスライド（3:45〜4:00）は再度ワイプで顔出し
- **マイク：** ピンマイクまたはヘッドセット使用。画面操作中のクリック音が入らないよう、マウスクリック音の少ない環境で収録する
- **画面解像度：** 1920×1080で収録し、フォントサイズ125%で文字の視認性を確保する
