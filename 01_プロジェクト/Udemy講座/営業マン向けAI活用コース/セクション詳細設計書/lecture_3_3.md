# レクチャー3.3 詳細設計書

## Webページの企業リストを整形 ― コピペ→Claude→Excel用タブ区切り出力

---

## ① レクチャーの位置づけ

レクチャー3.2では「PDFからの抽出がうまくいかないケース」と、そのリカバリー術を扱った。受講者は「PDFは万能じゃないんだな」という現実的な理解を持った状態でこのレクチャーに入る。本レクチャーでは、営業マンがもう1つ頻繁に直面する情報源——**Webページに掲載された企業リスト**——を扱い、「コピペ → Claudeに貼り付け → タブ区切りで出力 → Excelに貼るだけ」という最短ワークフローを完成させる。受講者が得る**1つのゴール**は「Webページ上の企業一覧を、ブラウザ拡張なし・特別なツールなしで、Excelにそのまま貼れるタブ区切りデータに変換できるようになること」である。次のレクチャー3.4「プロンプトの"型"を理解する」では、S3全体で使ったプロンプトを分解・解説するため、本レクチャーは「型の実例ストック」を1つ追加する橋渡しの役割も担う。

## ④ ライブ実演の操作手順

```
【実演パート名：Webページの企業リスト → タブ区切り変換 → Excel貼り付け】

＜操作手順＞

── ステップ1：Webページからテキストをコピーする ──
1. ブラウザのタブ1（架空Webページ「愛知県中小企業振興協会 会員企業一覧」）を表示する
2. ページをゆっくりスクロールし、企業情報が並んでいることを受講者に見せる（3秒）
3. 企業リスト部分の先頭（1社目の会社名）にカーソルを置く
4. Ctrl+A（全選択）でページ内のテキストを全選択する
5. Ctrl+C でコピーする
6. ※補足：「実務では必要な部分だけドラッグ選択してもOKです。今回は全選択で楽にやります」と一言添える

── ステップ2：Claudeにプロンプト＋コピペデータを送る ──
7. ブラウザのタブ2（Claudeの新規チャット画面）に切り替える
8. 入力欄をクリックする
9. 事前にテキストエディタに用意してあるプロンプトをコピー＆貼り付けする
   ────────────
   以下はWebページからコピーした企業一覧のテキストです。
   ここから次の5項目をタブ区切りで抽出してください。
   1行目はヘッダー行にしてください。

   【抽出する列】
   - 会社名
   - 代表者名
   - 住所
   - 電話番号（ハイフンは除く）
   - 業種

   ---
   ────────────
10. プロンプト末尾の「---」の直後にカーソルを移動し、改行を入れる
11. Ctrl+V でWebページからコピーしたテキストを貼り付ける
12. 入力欄にプロンプト＋企業データが一緒に入っていることを確認する
13. 「ぐちゃぐちゃに見えますが、大丈夫です」と一言添える
14. Enterキーで送信する

── ステップ3：出力結果を確認する ──
15. Claudeが出力を生成するのを待つ（5〜10秒）
16. 出力された結果をゆっくりスクロールして全体を見せる
17. 1行目（ヘッダー行）を指差し、「会社名・代表者名・住所・電話番号・業種、指示通りの5列ですね」と確認する
18. 2〜3行目のデータを指差し、「ハイフンが除かれている」ことを確認する

── ステップ4：Excelに貼り付ける ──
19. Claudeの出力の右上にある「コピー」ボタン（四角が重なったアイコン）をクリックする
20. ブラウザのタブ3（Excel 空のシート）に切り替える
21. セルA1をクリックして選択する
22. Ctrl+V で貼り付ける
23. 各列にデータがきれいに分かれて入ったことを確認する（セルB1が「代表者名」、C1が「住所」等）
24. 「タブ区切りで出力したおかげで、Excelが自動的に列を分けてくれました」と解説する

＜講師が強調すべきポイント＞
- 「特別なツールは一切使っていません。ブラウザとClaudeだけです。」
- 「"タブ区切りで"という一言がミソです。これがないとカンマ区切りになったり、表のまま出てきたりして、Excelに貼ったときに1つのセルに全部入ってしまいます。」
- 「"ハイフンは除く"のように、いらないものを先に指定しておくと、後でExcel上で消す手間が省けます。」

＜想定トラブルと対処＞
- コピペしたテキストにWebページのナビゲーション（「ホーム | お問い合わせ」等）が混入した場合：
  → 「余計な部分が入っても、Claudeは企業データだけを拾ってくれます。もし混ざっちゃったら、"企業情報だけ抽出して"と追加で指示すれば大丈夫です。」
- Claudeが表形式（Markdown表）で出力してしまった場合：
  → 「"タブ区切りで"と書いたのに表になっちゃうことがたまにあります。そのときは"表ではなくタブ区切りのプレーンテキストで出力し直してください"と送れば修正されます。」
- 15社のうち1〜2社のデータが抜けた場合：
  → 「たまに数社抜けることがあります。"全部で15社あるはずです。抜けがないか確認して"と送ると、Claudeが再チェックしてくれます。」
```

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示
- テンションは「同僚に便利技を教える」くらいのカジュアルさ。先生口調にならない。
- 「簡単でしょ？」ではなく「これだけで終わりです」と事実ベースで伝える。押し付けがましくしない。
- ライブ実演中は操作のテンポを落とし、「今ここを押しました」と実況を入れる。スマホの小さい画面でも操作箇所が伝わるようにする。

### NGワード

| NGワード | 言い換え |
|---------|---------|
| パースする／パース | 読み取る／整理する |
| タブ区切り（初出時に説明なしで使う） | 「タブ区切り、つまりExcelが列を分けてくれる形式」と初出時に必ず補足する |
| テキストデータ | 文字のデータ |
| プレーンテキスト | そのままの文字（飾りなしの文字） |
| クリップボード | コピーした内容（「クリップボードにコピーされました」→「コピーできました」） |
| CSS / HTML / DOM | 使用禁止（言い換え不要。言及自体をしない） |
| スクレイピング | 使用禁止（本レクチャーの手法はスクレイピングではなくコピペなので、言及自体をしない） |

### カメラ・マイク
- **顔出し：** 冒頭スライド（0:00〜0:25）のみワイプで顔出し。ライブ実演中（0:25〜2:45）は画面共有のみ（顔ワイプなし）で、操作画面を最大表示する。Before/Afterスライド（2:45〜3:00）は再びワイプで顔出し。
- **マイク：** 操作音（キーボード・マウスクリック）が入らないよう、マイクは口元から15cm以内に固定。

### 編集指示
- **Claudeの応答待ち時間（1:50〜2:00付近）：** 実際に5秒以上かかった場合は、2〜3秒にカット編集する。画面右下に「※ 実際の待ち時間：約○秒」のテロップを入れる。
- **Excel貼り付け後（2:25〜2:45）：** データがきれいに列に分かれた瞬間に、0.5秒の効果音（「ピコン」等の達成音）を入れてもよい。派手にしすぎない。
- **テロップ：** プロンプトの「タブ区切りで」の部分に黄色のハイライトテロップを重ねる（2:10付近で「ここがポイント！→"タブ区切りで"」と表示）。

### 前後のレクチャーとの接続
- **冒頭（0:00）：** 「前回はPDFからのデータ抽出をやりました」と一言触れるだけでよい。前回の内容を要約する必要はない。
- **末尾（2:55〜3:00）：** 「次のレクチャーでは、今使ったプロンプトの"なぜこう書くと精度が上がるのか"を解説します。応用力がグッと上がるので、ぜひ続けて見てください。」と予告し、3.4への期待を持たせる。
