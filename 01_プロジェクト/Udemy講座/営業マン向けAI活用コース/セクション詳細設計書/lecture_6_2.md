# レクチャー6.2 詳細設計書
## 業種別DM一括生成 ― 製造業・IT・小売の3パターンを一度に作る

---

## ① レクチャーの位置づけ

前のレクチャー6.1で、受講者は「AIにDMの基本構成を教えてもらう」という体験を通じて、件名・冒頭・本文・CTAの"型"を手に入れている。本レクチャーでは、その型を使って**3業種分のDMを一度の指示で量産する**ライブ実演を行い、受講者の最大の動機である「量産したい」に正面から応える。ここで「1回の指示で複数パターンが出る」という成功体験を積ませることで、次のレクチャー6.3「1社ごとのパーソナライズ」で「量産＋質」の両立へと自然に橋渡しする。本レクチャーはS6の中核であり、「AIで営業DMを量産できる」というコース全体の最大の売りの一つを体現するパートである。

## ④ ライブ実演の操作手順

```
【実演パート名：業種別DM一括生成】

＜操作手順＞
1. メモ帳（またはテキストエディタ）を画面に表示する
   → 事前準備した営業リスト（9社分）が表示されている状態を受講者に見せる
2. ブラウザに切り替え、Claudeのチャット画面を開く
   → 新規チャット画面であることを確認（前のチャット履歴が見えない状態）
3. メモ帳に切り替え、事前準備したプロンプト全文を選択（Ctrl+A）してコピー（Ctrl+C）
4. Claudeのチャット画面に戻り、入力欄をクリック
5. 入力欄にプロンプト全文を貼り付け（Ctrl+V）
6. 貼り付けた内容を3秒ほど画面に映す（受講者にプロンプトの構造を見せるため）
   → 「"業種ごとに1パターンずつ"と書いているのがポイントです」と指差し
7. Enterキーで送信
8. Claudeの応答が生成される画面をそのまま映す
   → 応答生成に10秒以上かかる場合は、編集で早送り処理
9. 応答が完了したら、まず画面最上部（製造業DM）までスクロール
10. 製造業DMの件名を指差し → 冒頭を指差し → 本文を指差し → CTAを指差し
11. 下にスクロールし、IT企業向けDMを画面に映す
12. 件名のキーワードの違い（「コスト削減」→「人材不足」等）を指差して比較
13. さらに下にスクロールし、小売業向けDMを画面に映す
14. 3パターン全体が出力されたことを確認するジェスチャー

＜講師が強調すべきポイント＞
- 「1回貼り付けて送信しただけです。手で3パターン書いたら1時間半。それが2分かかっていません。」
- 「大事なのは、"業種の特徴を踏まえて書き分けて"と指示したこと。これを言わないと全部同じ文面になります。」
- 「"なんとなく良い感じ"ではなく、なぜその訴求にしたかの解説まで出てきています。」

＜想定トラブルと対処＞
- 3パターンではなく1パターンしか出力されなかった場合：
  → 「"残りの2業種も同じ形式で作成してください"と追加で指示を送る」と実演
- DM文面が長すぎる場合（本文が10行以上）：
  → 「"本文は5行以内に短くしてください"と追加指示を送る。AIは指示すれば直してくれます」と実演
- 業種ごとの書き分けが弱い（似たような文面になっている）場合：
  → 「"製造業にはコスト削減、ITには人材不足、小売には集客を軸にもっと明確に書き分けて"と具体的に追加指示する」と説明
```

## ⑥ 講師メモ（撮影時の注意点）

### トーンの指示
- **テンション：やや高め、テンポよく。** このレクチャーは「量産できる快感」を届けるパート。驚きとスピード感を重視し、「すごくないですか？」「これだけで終わりです」のようなリアクションを自然に挟む。
- **語り口：** 「同僚に教える」くらいのカジュアルさ。ただし受講者を見下す表現（「簡単でしょ？」「こんなこともできないの？」）は絶対に使わない。
- **間の取り方：** 3パターンの出力結果を見せる場面では、各DMを2〜3秒ずつ画面に映して「読む間」を取る。早送りしすぎない。

### NGワード

| NGワード | 言い換え |
|---------|---------|
| API | （このレクチャーでは使う場面なし） |
| プロンプトエンジニアリング | 「指示の書き方」「指示のコツ」 |
| テンプレートリテラル | （使わない） |
| パラメータ | 「条件」「設定」 |
| トークン | （使わない） |
| バッチ処理 | 「一括で作る」「まとめて作る」 |
| ペルソナ設定 | 「役割を伝える」 |
| イテレーション | 「やり直し」「修正指示」 |

### カメラ・マイク
- **顔出し：** 冒頭スライド（0:00〜0:15）とまとめ（2:50〜3:00）のみワイプで顔出し。ライブ実演中は画面共有のみ（操作が見やすいように）
- **マイク：** 外部マイク使用。操作中もナレーションは途切れさせない

### 編集指示
- **0:45〜1:10（プロンプト貼り付け）：** 貼り付け後、プロンプトの重要部分（「業種ごとに1パターンずつ」「各業種の特徴を踏まえて書き分けること」）にテロップで下線 or ハイライトを追加
- **1:20〜1:50（Claude応答生成中）：** 応答生成が10秒以上の場合は「×2 早送り」のテロップを右上に表示して早送り。5秒以内で完了した場合はカットなし
- **1:50〜2:50（出力結果確認）：** 各業種DMの件名部分にテロップで「製造業向け」「IT企業向け」「小売業向け」のラベルを画面右上に表示
- **全体：** BGMは軽快なもの（テンポ120BPM程度）。ライブ実演中は音量を下げる

### 前後のレクチャーとの接続
- **冒頭（6.1からの接続）：** 「前回、AIにDMの"型"を教えてもらいましたよね」と一言で振り返る。6.1の内容を繰り返さない
- **末尾（6.3への予告）：** 「でも、"もっと反応率を上げたい"と思いませんか？　次のレクチャーでは、相手の会社のホームページを踏まえて、1社ずつDMをカスタマイズする方法をお見せします」と、量産→質への自然な橋渡しを行う
