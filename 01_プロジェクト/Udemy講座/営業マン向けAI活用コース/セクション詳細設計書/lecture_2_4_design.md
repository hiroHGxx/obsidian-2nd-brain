# レクチャー2.4 詳細設計書

## 社内でAI利用許可を勝ち取る ― 上司・情シス向け説明テンプレート

**レクチャーID:** 2.4  
**レクチャー時間:** 2分  
**セクション:** S2: AIを安心して使い始める ― アカウント作成から社内許可まで

---

## ① レクチャーの位置づけ

前のレクチャー2.3で、受講者は「Claudeは入力データを学習に使わない」「ChatGPTは設定でオフにできる」というセキュリティの事実を理解した状態にある。本レクチャーでは、その知識を"自分の言葉で上司や情シスに説明できる状態"に変換することが役割である。受講者の多くが抱える「技術的には安全だとわかったが、社内にどう説明すればいいかわからない」という行動障壁を、メールテンプレートとチェックリストという"そのまま使える武器"で突破させる。これにより、次のレクチャー2.5（プロンプトの基本の型）以降の実践パートに、心理的にもルール的にも安心して入れる状態を作る。

## ⑥ 講師メモ（撮影時の注意点）

**トーンの指示：**  
このレクチャーのテーマは「あなたの代わりにやっておいたから、安心して」。受講者が感じている"上司への説明が面倒・不安"という心理に共感しつつ、「テンプレートを渡すだけで解決する」という軽さを全面に出す。声のトーンは明るく、テンポよく。"許可を取る"という重たい行為を、"メール1通送るだけ"という軽い行動に変換するのが最大のポイント。

**NGワード：**

| NGワード | 言い換え |
|---------|---------|
| データポリシー | データの扱い方のルール |
| コンプライアンス | 社内ルール |
| セキュリティポリシー | 安全のための取り決め |
| API | （このレクチャーでは言及不要） |
| オプトアウト | 学習に使わない設定 |
| リスクアセスメント | 安全かどうかの確認 |
| ガバナンス | 管理の仕組み |

**カメラ・マイク：**  
画面共有のみ（スライド表示＋画面収録）。顔出し不要。マイクはナレーション収録。スライド切り替え時に無音の間が生じないよう、セリフは途切れなく話し続ける。

**編集指示：**

- 画面収録パート（1:35〜1:50）は事前収録を2倍速に早送り編集する
- 「リソース」ボタンとファイル名の位置に赤い丸囲みアニメーションをポストプロダクションで追加
- スライド3のメールテンプレートプレビューは、テキストが1行ずつフェードインするアニメーション推奨（講師のセリフに合わせて）
- 全体の尺が2分00秒を超える場合、スライド間の切り替え時間を詰めて調整する（セリフは削らない）

**前後のレクチャーとの接続：**

- **冒頭（2.3からの接続）：** 「前回のレクチャーで、Claudeは入力データを学習に使わないこと、ChatGPTもオフにできることがわかりました」と1文で振り返る。2.3の内容を前提知識として使うため、必ず触れる。
- **末尾（2.5への橋渡し）：** 「アカウント✓ 安全性✓ 社内許可✓ ——準備は万端。次は、AIに"伝わる指示の出し方"を覚えます」とS2全体の完了感を出しつつ、次のレクチャー2.5（プロンプトの3つの型）への期待を作る。2.5はS3以降の実践で繰り返し使う"共通言語"を仕込むレクチャーなので、「この型さえ覚えれば、あとは全部この応用です」という予告が効果的。
