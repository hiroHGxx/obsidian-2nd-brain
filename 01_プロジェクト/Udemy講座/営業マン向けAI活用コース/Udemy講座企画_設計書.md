# Udemy講座企画：営業マン向けAI活用コース ― 引き継ぎドキュメント

> 作成日：2026年2月7日｜全4セッションの議論を集約

## 1. プロジェクト概要

### 1-1. 出発点

営業担当者から寄せられた以下の相談をUdemy講座化するプロジェクト：

> 「ホームページの企業リストやPDFを毎回手作業でExcelにコピーして、営業リストに整えたり、自分の営業所に近いところから順番に回るための方法を考えたり（地図で同心円を描いたり）、DMを送ったりしてるのを楽にできないか。DM教材も買ったりしてるけど、AIでこのあたりって楽にできそうだよね」

### 1-2. 講座の核心メッセージ

**「月額○万円のSFAツールを導入しなくても、今あるデータをClaudeに渡すだけで、手作業の大半を置き換えられます」**

### 1-3. 講座が提供する4つの自動化

| #   | 手作業（Before）            | AI活用（After）                           |
| --- | ---------------------- | ------------------------------------- |
| ❶   | HP・PDFから企業情報をExcelに手入力 | Claudeにアップロード/コピペ → 表形式で一発抽出          |
| ❷   | Excelで営業リストとして手動整形     | 「重複排除・業種フィルタ・優先度ランク付け」を日本語指示で自動化      |
| ❸   | 地図に同心円を描いて訪問順を決定       | Claudeでエリア分類 → Googleマイマップにインポートで視覚確認 |
| ❹   | DM教材を見ながら1件ずつ文面を作成     | 業種別DM一括生成 → 1社ごとのパーソナライズも可能           |

## 2. 戦略的な意思決定（仮説検証プロセスで確定したもの）

### 2-1. 市場ポジショニング

**発見：「営業リスト作成×AI」特化のUdemy日本語講座は2026年2月時点でゼロ（ブルーオーシャン）**

ただし、以下の構造的リスクを認識した上での判断：
- 「ChatGPT × ビジネス効率化」汎用講座は飽和状態（矢野哲平氏8,000名超、ウミノ氏フォロワー2.3万人など実績講師が上位独占）
- 「Claude AI」日本語検索ボリュームは「ChatGPT」の10分の1以下
- Udemyで「営業リスト」を直接検索する人は少ない可能性あり

**→ 採用した戦略：** タイトルは広く張り（「営業効率化」「ChatGPT対応」）、中身は狭く深く（リスト作成特化）にすることで、検索流入と差別化を両立する

### 2-2. 競合の定義

**Claudeの競合はSaaSツールではなく、「今の手作業のまま」という選択肢。**

営業リスト作成SaaS（IZANAMI、Musubu、APOLLO SALES等）は月額数万円。ターゲットの営業マンにはその予算も決裁権もない。講座は「SaaSの代替」ではなく「手作業の代替」として位置づける。

### 2-3. ターゲットの二層構造と段階戦略

| フェーズ | ターゲット | 設計方針 |
|---------|-----------|---------|
| **フェーズ1（本人型）** | 営業マン自身が「楽になりたい」と思って個人購入 | 短い・すぐ使える・移動中に見れる。1レクチャー5分以内、スマホ視聴前提 |
| **フェーズ2（推薦型）** | マネージャーや経営者が「チームに見せたい」と思って法人導入 | レビュー蓄積後、Udemy Business申請。マネージャー向けガイド追加 |

### 2-4. 購買心理の理解

営業マンの学習動機はエンジニアと根本的に異なる：
- ✗「新しい技術を学びたい」（スキルアップ志向）
- ○「今やっている面倒な作業を楽にしたい」（即効性志向）

**→ 講座内の語り口は「学び」ではなく「解放」。**

### 2-5. 相談者の深層心理（表面的な悩みの裏にあるもの）

| 感情 | 推測される心理 | 講座での対処 |
|------|-------------|------------|
| 孤独感 | 「他の会社ではどうやってるんだろう」。効率化の情報交換ができない | コミュニティ・仲間の存在を示唆 |
| 焦燥感 | DM教材を自費で買っている。努力はしている。でも成果が見えない | 「教材なしでもClaude+あなたの経験で回せる」 |
| 無力感 | SFAツール導入の決裁権がない。会社はDXに投資してくれない | **「会社を待たなくても、一人で始められる」** |
| 不安感 | AIが話題だけど自分は使えない。取り残される不安 | 「今日の帰りの電車で1レクチャー見るだけでいい」 |

### 2-6. ROI定量化

- 営業マンのリスト作成作業：週3〜5時間 → Claudeで30分〜1時間に短縮
- 週あたり時短効果：2〜4時間（月8〜16時間）
- 営業マンの平均時給2,500円換算：**月2万〜4万円相当の人件費削減**
- Claude Pro月額：約3,000円
- **→ 月3千円で月2〜4万円削減のROI**

## 3. 技術的な設計判断

### 3-1. Claudeの技術的な得意/不得意（正直に教える方針）

| データソース | Claudeの精度 | 講座での教え方 |
|-------------|------------|-------------|
| テキストベースPDF | ◎ 高精度 | メイン実演に使用 |
| スキャンPDF | △ 不安定 | 「OCRアプリで先に変換→Claudeに渡す」の2段階 |
| 複雑レイアウトPDF | △ 列が崩れる | 「列名を指定して再指示する」リカバリー方法 |
| Webページの表 | ○ コピペで整形可能 | ブラウザ拡張なしの最小手順 |
| 紙の名刺・チラシ | △ 完璧ではない | 「たたき台としては十分」と正直に伝える |

### 3-2. 距離計算の正直な限界

Claudeは正確な距離計算ができない。「名古屋市中区と北区どちらが近いか」はわかるが、「豊田市挙母町とみよし市三好町の距離差」は出せない。

**→ 採用したワークフロー：**
1. **Claude：** 住所リストを市区町村・エリアでグループ分け（Claudeが得意）
2. **Googleマイマップ：** CSVインポートで地図上にピン表示（視覚確認）
3. **Googleマップ：** 当日の訪問先を入力→最適ルート表示

**「Claudeで"ざっくり仕分け" → Googleマップで"正確に確認"」という使い分けを正直に教える。**

### 3-3. Claude × ChatGPT対応方針

メインはClaudeで実演しつつ、「ChatGPTでも同じプロンプトで同様の結果が出ます」と補足。タイトルに「ChatGPT対応」を入れて検索流入を拡大。

## 4. 確定したタイトル・副タイトル・コース説明

### 4-1. タイトル（パターン5を採用）

> **営業効率化の決定版！Claude×ChatGPT対応｜リスト作成・DM一括生成・訪問計画をAIで自動化する実践講座【プロンプトテンプレ付】**

（60.5文字 ※Udemy上限60文字、半角英数0.5文字カウントルール適用）

**戦略意図：** 「Claude×ChatGPT対応」で両検索吸収。「プロンプトテンプレ付」は高コンバージョンワード。「営業効率化」先頭でマネージャー層・法人購入にもリーチ。

### 4-2. 副タイトル（パターン3を採用）

> **毎週3〜5時間かかっていた営業リスト作成の手作業をわずか30分に短縮できる｜月3千円のClaude Proで月2〜4万円相当の人件費を削減するROI実証済みの方法論｜PDF・Web・Excelデータの取り込みから重複排除・優先度ランク付け・エリア分類・DM生成まで完全収録！**

（120.5文字）

**戦略意図：** 定量的ROI（月3千円→月2〜4万円削減）で費用対効果を明確化。タイトルに入らなかった「PDF」「Excel」「Web」の具体的データ形式を配置。

### 4-3. コース説明文（約500文字）

> 毎週何時間もかけてExcelに企業情報を手打ちし、地図を見ながら訪問ルートを考え、DMの文面に頭を悩ませていませんか？
>
> この講座では、その「めんどくさい」を生成AIで一気に解決します。
>
> 月額○万円のSFAツールを導入しなくても、手持ちのPDFやWebページのデータをClaudeに渡すだけで、営業リストの整形・重複排除・優先度ランク付け・エリア別分類・業種別DM一括生成まで、すべて日本語の指示だけで完了します。
>
> 本講座の特徴は「成功例だけ見せない」こと。PDFの読み取りが崩れたときの対処法や、AIに社内データを渡す際のセキュリティ対策まで、現場で本当に必要な知識をセットで教えます。
>
> 全レクチャー5分以内。移動中のスマホ視聴を前提に設計しているので、忙しい営業マンでも無理なく学べます。Claude・ChatGPTの両方で使えるプロンプトテンプレート30選とA4手順チェックシートを特典としてお付けしています。会社がAIツールを導入してくれるのを待つ必要はありません。明日の朝、最初の1回を試してみてください。

## 5. Udemy申請用情報（目標・要件・ターゲット）

### 5-1. Udemy学習目標（7項目）
1. 週3〜5時間かかっていた営業リスト作成作業を30分〜1時間に短縮し、月2〜4万円相当の人件費削減を実現できるようになる
2. PDF・Webページ・Excelから企業情報を一発抽出し、手入力作業をゼロにできるようになる（テキストPDF・スキャンPDF・複雑レイアウトそれぞれの対処法を習得）
3. 業種・規模・エリア別に刺さるDMを一括生成し、1社ごとのパーソナライズまで自動化できるようになる（DM教材なしでも高反応率のアプローチ文を量産）
4. 重複排除・業種フィルタ・優先度ランク付けを日本語指示だけで完了させ、Excel関数やマクロなしで「戦略的な営業リスト」を仕上げられるようになる
5. 営業所を起点としたエリア分類からGoogleマイマップ連携まで習得し、「地図で同心円を描く」手作業を自動化して効率的な訪問計画を立てられるようになる
6. 社内のセキュリティポリシーに沿ったAI活用方法を身につけ、上司や情報システム部門の許可を得るための具体的な手順を実践できるようになる
7. プロンプトの「型」を理解して応用力を身につけ、この講座で扱わない業務（見積書作成・議事録整理・競合分析など）にもClaude/ChatGPTを自力で活用できるようになる

### 5-2. Udemy受講要件・前提条件（5項目）
1. 必須: 営業リスト作成の実務経験（PDF・Webページから企業情報を手作業でExcelにまとめた経験があれば十分です）
2. 必須: インターネット接続可能なPC、タブレット、またはスマートフォン（講座視聴と実践の両方に使用します）
3. 必須: Microsoft Excel（または互換ソフト）とWebブラウザ（Google Chrome/Edge/Safari等）の基本操作ができること
4. 推奨: 生成AI（Claude・ChatGPT）の利用経験は一切不要です（アカウント作成から丁寧に解説します）
5. 推奨: Claude Pro（月額3,000円程度）またはChatGPT Plus（月額20ドル）の契約（無料プランでも受講・実践可能ですが、有料プランでより快適に学べます）

### 5-3. Udemy想定学習者（5項目）
1. 毎週3〜5時間をPDFやWebページからの企業情報の手入力作業に費やし、「もっと効率的な方法があるはず」と感じている法人営業・新規開拓担当者
2. DM作成に毎回頭を悩ませ、高額なDM教材を購入したものの成果が見えず、「もっと楽に、もっと刺さる文面を量産したい」と考えている営業マン
3. 地図を見ながら訪問ルートを手作業で計画する時間を削減し、限られた営業時間で1件でも多くの顧客訪問を実現したいフィールドセールス担当者
4. 営業チームの生産性向上を求められているが、高額なSFAツール導入の予算がなく、「低コストで今すぐ始められる営業効率化の方法」を探している営業マネージャー
5. AIやChatGPTが話題になっているが「自分には難しそう」「何から始めればいいかわからない」と感じており、営業実務で今日から使える具体的な活用法を知りたい営業職全般

### 6-2. カリキュラム詳細設計の意図

#### ★ 追加・変更レクチャーとその理由

**追加1：レクチャー 2.3「『会社のデータ、AIに取られない？』― 3分でわかる安心設定」（3分）**
元の設計ではセキュリティ関連がS7（コース終盤）に配置されていたが、S3〜S6の実践中ずっと「これ、会社のデータ入れて大丈夫なのかな…」というモヤモヤを抱えたまま進むことになる構造上の問題があった。S2の準備編に統合することで、実践パートに入る前に不安をすべて取り除き、S3以降は純粋に手を動かすことに集中できるようにした。また、元の設計では不足していた「Claudeはそもそも入力データを学習に使わない」という事実と「ChatGPTの学習除外設定の具体的手順」を新規追加し、ツールごとの安心材料を明確にした。

**追加2：レクチャー 2.4「社内でAI利用許可を勝ち取る ― 上司・情シス向け説明テンプレート」（2分）**
元のS7.2から移動。2.3でデータの安全性を理解した直後に「だから安全です」と上司に説明できる流れを作ることで、「学び」→「理解」→「行動」の一貫したストーリーを構築。行動障壁分析で「発生確率：非常に高」と評価された「顧客データは機密、AIに入れていいのか」を、コース終盤ではなく序盤で正面から潰す設計に変更。

**追加3：レクチャー 2.5「AIへの指示の基本 ― "伝わるプロンプト"の3つの型」（1分）**
元のレクチャー2.3と同内容を2.5に移動。プロンプトの書き方がS3-4で初めて登場するのはAI未経験者には唐突なため、S2の準備編で「①役割を与える ②具体的に指示する ③出力形式を指定する」の3つの型を1分で"予告"しておくことで、S3以降の実践がスムーズに理解でき、「プロンプトの型」がコース全体を貫く共通フレームワークとして機能する。

**廃止：旧S7「これだけは知っておくべきAIセキュリティ」**
S7の内容はS2.3およびS2.4に統合・強化のうえ移動したため、独立セクションとしては廃止。旧S8「明日の朝から始める第一歩」をS7に繰り上げ、全体を7セクション構成に変更。

## 8. SEO最適化：キーワード戦略

### 8-1. 検索ボリュームの現実

- 「Claude AI」は「ChatGPT」の10分の1以下 → Claude単独は危険
- 「営業リスト作成 AI」はGoogle検索で大量の記事（需要あり）だが、Udemy内での検索は少ない
- 「生成AI 営業 効率化」「ChatGPT 営業 活用」は高ボリューム

### 8-2. 採用キーワード一覧

| カテゴリ | キーワード | 採用理由 |
|---------|-----------|---------|
| ツール名 | ChatGPT / 生成AI | 検索ボリューム確保（Claude単独では不足） |
| 業務系 | 営業効率化 / 新規開拓 / 営業リスト作成 | 直接的な課題ワード |
| 訴求系 | 時短 / 自動化 | 営業マンの「今すぐ楽に」動機に直結 |
| スキル系 | プロンプト / テンプレート | 高コンバージョンワード |
| 対象者系 | 初心者 | AI未経験者のハードル下げ |
| データ系 | Excel / PDF / Web | 実務で扱うデータ形式。具体性→信頼感 |
| 差別化 | 月額ツール不要 | SaaSとの差別化。予算がない個人営業マンへ訴求 |
| トレンド | 営業DX / AI営業 | 法人購入（Udemy Business）を見据えたワード |

### 8-3. Udemy文字数ルール

- タイトル：60文字上限
- 副タイトル：120文字上限
- カウント方法：全角1文字、半角英数0.5文字
